ScoringEngine 集成点选择

MockDataStore 不排序后，排名逻辑放在哪里？

方案 B1: 在搜索 Tool 中排序
AgentLoop
调用 toolRegistry.execute(req)
▼
ToolRegistry
dispatch 到具体 Tool
▼
🆕 RestaurantSearchTool
① 调 MockDataStore.search() → 过滤列表
② 调 scoringEngine.rank() → 排序
③ 返回排序后 JSON
▼
MockDataStore — 纯过滤，不排序
⚡ 问题：Tool 没有 SessionContext
需要修改 Tool 接口传入场景信息
方案 B2: 在 AgentLoop 中排序
AgentLoop
① 调 toolRegistry.execute(req) → 原始 JSON
② 检测是搜索类 Tool → 调 scoringEngine.rankJson()
③ 排序后 JSON → 喂给 LLM
▼
ToolRegistry
dispatch（不变）
▼
MockDataStore — 纯过滤，不排序
⚡ 问题：AgentLoop 需要解析 Tool JSON
AgentLoop 耦合了搜索结果的数据结构

还有第三个选择

方案 B3: ScoringEngine 通过 ToolRegistry 注入，在 Tool 层使用 ThreadLocal SessionContext
ToolRegistry.execute() 调用前
SessionContextHolder.set(ctx)
SearchTool 内部
SessionContext ctx = SessionContextHolder.get()
调 MockDataStore.search() → 调 ScoringEngine.rank()
ToolRegistry.execute() 返回后
SessionContextHolder.clear()
✅ Tool 接口不变   ✅ AgentLoop 不解耦   ✅ MockDataStore 不排序
⚠️ 但引入了 ThreadLocal（Spring 单线程模型下安全，WebSocket 需注意）
